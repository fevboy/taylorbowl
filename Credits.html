Credits form the basis of the TaylorBowl economy. They are the most accessible and commonly used currency and the majority of actions related to currency involve credits.

You may use the command !creds to check the number of credits you have.
To earn credits, there are ways involving no risk and ways involving a risk. For those with risk, the expected value of using them is typically around 1, meaning that repeating them in a random manner for a long time will typically not result in any gains or losses.
Voting
For every vote in a TaylorBowl, the user receives 10 credits. This number is increased by your Streak; every 7 Streak increases the number of credits received by 10.

If you are a Server Booster, Kintsugi, or both, you receive extra credits from voting. Server Boosters receive 1.25x of the standard, rounded up. Kintsugi receive 2x. Both of these can stack, leading to a 2.5x multiplier.

You can see the number of credits you will receive from voting in a bowl by using the command !streak. It will display all your streak-related information.

You can also use the EsteBot commands /streakcurrent to calculate the number of credits obtained from your next vote at a given streak; /streaknext to see the next streak where your credits obtained from voting increase; and /streakcumulative to determine the total credits earned when you streak from a certain number to another.

Note: /streakcumulative is not fully accurate for users who are Server Boosters but not Kintsugi, on average underestimating the number by 7 credits every 28 bowls, though this is a negligible amount.
Minigames
There are 7 minigames that can give you credits, 6 of which involve a risk.

Farm is a farm simulator minigame that allows you to plant credits. At random intervals, they can grow by a certain percentage. They also have a smaller chance to be destroyed.

The Commons is a variation of the Farm minigame, where credits are collectively planted and redistributed each time they grow. If a Bot Restart occurs, all credits in the Commons are destroyed.

Ballpit is a minigame involving probability. 100 balls of different colours are randomly arranged, and the user may place a bet about which balls will appear first in the sequence. Depending on the rarity of the ball, the user receives different amounts of credits if they win. If they lose, all credits bet disappear.

Risk is a die-based minigame. A 20-sided die is rolled. If the number rolled is between 2 and 20, the user receives that number of credits. If the number rolled is 1, the user loses 209 credits. The user can also multiply all returns and losses by specifying an integer multiplier.

Anti-Risk is a die-based minigame. A 20-sided die is rolled. If the number rolled is between 1 and 19, the user loses that number of credits. If the number rolled is 20, the user receives 190 credits. The user can also multiply all returns and losses by specifying an integer multiplier.

Yahtzee is a dice-based minigame. Five 6-sided dice are rolled, and each roll costs 10 credits. Depending on the combination of numbers shown, the user has a chance of earning credits. Every Bot Cycle, all users get a number of free Yahtzee rolls that do not cost credits.

LyricRace is a race-style minigame. No credits are required to play. A set of 13 prompts related to Taylor Swift's songs and albums are given, and the user is asked to answer correctly as fast as possible. Depending on how long the user took to answer all 13 questions, they will receive credits at the end of the race.
Potentia
The command !summon can be used to turn Potentia into credits. Each Potentia is converted directly into credits, worth 0.05% of all Active Credits.
Credits have a number of uses, notably being used to perform certain actions or commands.
Forcing Bowls
Main article: Forcing & Queueing
To force or queue a TaylorBowl at a specific Tier, length, or with a specific track, credits must be used. Depending on those three factors, it may be more or less expensive.
Bribing
Main article: Bribing
If a TaylorBowl enters Tiebreaker, credits are the only way to influence the final result of the bowl. The command !bribe amount can be used to bribe for a song. The more credits used to bribe for a given song, the more likely it is to win.
Autovote
Main article: Autovote
To autovote, credits are required depending on the Autovote Streak of the user.
User Score
Main article: User Score
Spent Credits count towards a user's score. First, each spent credit is worth 1 score, but they can only make up a maximum of half of a user's score before Lyric IQ. All spent credits above this threshold are worth less.
Transfers
Credits can be transferred from one user to another. At least 250 credits must be transferred, and a tax is incurred when doing so. This is done using the command !transfer amount @user. This command is unable to be used during the last 20 seconds of a bowl to prevent errors.
Ephemera
Credits can be used to create Ephemera or gain Ephemera Levels. The command !condense can be used to turn credits into Ephemera. The command !exceed can be used to increase your Ephemera Level.
Combat
Main article: Combat
To Discover a new song, which adds it to the user's Chapter, the command !discover can be used. The cost to discover starts at 100 credits, and increases by 100 credits for every song the user currently owns, given by the formula:

$$\text{Cost}=(\text{Current number of songs}+1)\cdot100$$

To Train a song, the command !train can be used. This increases the Level of a currently-owned song by 1. The cost to train starts at 10 credits, and increases by 10 credits for every additional level of the song. This cost is multiplied by the Slot Number of a song. The cost is given by:

$$\text{Cost}=\text{Current level}\cdot\text{Slot number}\cdot10$$

Credits are subject to three taxes. Generally, they aim to encourage usage of credits and being active.
Wealth Tax
Wealth tax is applied to all users with 10 000 credits and above. It is applied every 50 TaylorBowls, removing a certain percentage of users total credits, or their on-hand and planted credits added together. The percentage taxed increases linearly based on the number of credits you have, by 0.5% every 100 000 credits, and reaching a maximum of 50%. It is given by the formula:

$$\text{Tax percentage}=\text{Number of credits}\cdot0.000\,005\%$$

Since the wealth tax is a percentage, the amount taxed increases quadratically based on your credits. That is, if your credits increase by \(n\) times, your wealth tax increases by \(n^{2}\) times. For example, 100 000 credits incur a tax of 500, 200 000 credits incur a tax of 2000, and 400 000 credits incur a tax of 8000. The number of credits taxed is given by the formula:

$$\text{Tax}=\cfrac{(\text{Number of credits})^{2}}{20\,000\,000}$$

You may use the command !owed to see the number of credits you owe due to wealth tax, and the command !taxes to see the total number of credits the server owes due to wealth tax, the last taxation, and the next taxation.
Transfer Tax
Transfer tax is applied when credits are transferred. It is given by the formula:

$$\text{Tax}=\text{max}\left( 8\cdot\text{log}_{10}(\text{Credits transferred})^{2},\, 0.025\cdot(\text{Credits transferred}) \right)$$

Since, for smaller numbers of credits, the logarithmic function is greater than the linear one, transfer tax percentage is higher for smaller numbers of credits, decreasing as the amount increases. However, once transfers are at least 4201 credits, the percentage value stabilises at 2.5%.

You may use the command !transfercost to determine how many credits will be taxed for a transfer of a given size. You may use the EsteBot command /wantedtransfer to determine how many credits must be transferred for a desired number of credits to be received.
Inactivity Decay
Inactivity Decay is applied to “inactive” users. Users are classified as inactive if they have not voted in a bowl during the current season, or if they are not a member of a server with TaylorBowl. Like wealth tax, it is applied every 50 bowls. The tax is always 0.25% of a user's total credits or 1 credit, whichever is greater. It is given by the formula:

$$\text{Tax}=\text{max} \left( 0.0025\cdot(\text{Total credits}),\, 1 \right)$$

Credits affect prices of various commands to respond to changes in the server's overall wealth.
Base Price
Main article: Base Price
The Base Price determines the cost of forcing a bowl. One of its components is 0.0005% of Active Credits, which refer to the total number of credits of all active users.
Condensing
Main article: Ephemera/Obtaining
The number of credits used to create Ephemera from credits by condensing is based on both the condensor's credits and active credits. The amount needed is the largest of: 0.075% of active credits, 1.5% of the condensor's total credits, or 1989 credits. It is given by the formula:

$$\mathrm{Cost}=\mathrm{max} \left(0.00075\cdot(\text{Active credits}),\, 0.015\cdot(\text{Total credits}),\, 1989 \right)$$

You may use the command !ephemera to check how much it would cost for you to condense.
Exceeding
Main article: Ephemera/Ephemera Level
The number of credits used to increase a user's Ephemera Level using the command !exceed is determined by Active Credits. The cost to exceed is 0.2% of Active Credits, multiplied by the resultant Ephemera Level, or the user's current Ephemera Level plus 1. It is given by the formula:

$$\mathrm{Cost}=0.002\cdot(\mathrm{Active\,credits})\cdot(\mathrm{Resultant\,Level})=0.002\cdot(\mathrm{Active\,credits})\cdot(\mathrm{Current\,Level}+1)$$

You may use the command !ephemeralevel to check how much your next exceed would cost.
List of Commands
Command Description
!baseprice Displays the Base Price and its components.
!bribe amount Bribes the specified number of credits for the user's choice.
amount is a mandatory parameter that refers to the number of credits bribed.
!creds user Displays a user's credits.
user is an optional parameter, defaults to command user.
!discover Discovers a song in Combat.
!ephemera Displays information about Ephemera and Invocations, if applicable.
!ephemeralevel Displays information about a user's Ephemera Level.
!exceed Increases the user's Ephemera Level.
!owed user Displays the taxes that a user will pay at the next taxation.
user is an optional parameter, defaults to command user.
!streak user Displays current Streak.
user is an optional parameter, defaults to command user.
/streakcumulative Estimates the credits earned from voting from one Streak to a higher one.
/streakcurrent Shows the credits earned from a bowl at a given Streak.
/streaknext Shows the next Streak where credits earned increases.
!summon amount Converts a specified number of Potentia into credits.
amount is an optional parameter that refers to the number of Potentia converted, omission shows a user's available Potentia.
!taxes Displays the total taxes paid.
!train number Trains a user's specified song in Combat.
number is a mandatory parameter that refers to the song's Slot Number.
!transfer amount @user Transfers a specified number of credits to a specified user.
amount is a mandatory parameter that refers to the number of credits transferred.
@user is a mandatory parameter that refers to the user credits are transferred to.
!transfercost amount Displays the Transfer Tax on a specified number of credits.
amount is a mandatory paramter that refers to the number of credits.
/wantedtransfer Calculates the number of credits that must be transferred to overcome the Transfer Tax.

List of Terms
Term Definition
Active Credits The combined total credits of all active users.
Base Price A variable number that determines prices across TaylorBowl.
Expected Value The average outcome of a probability-based event.
Inactive User A user who has not voted in a Bowl since the start of a Season, or is not in any TaylorBowl-enabled servers.
Total Credits A user's combined on-hand and planted credits.
